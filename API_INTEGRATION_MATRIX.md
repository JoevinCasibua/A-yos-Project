# API Integration Matrix

Status values describe the implementation after the three-interface integration work. “E2E pending” means the application path is connected but lacks an authenticated fixture/device acceptance run; it is not production-complete evidence.

| UI page/feature        | Role         | Component/action                                   | Backend interface and method                                                             | Request                                                         | Expected response                                       | Authorization                                  | Loading/success/error behavior                                            | Baseline                                    | Test evidence                  |

| Register               | User, Worker | Submit account form                                | Supabase Auth `signUp`                                                                   | Email, password, fixed role, name, E.164 mobile, accepted Terms | Pending Auth identity and application account           | Anonymous; provisioning rejects Admin metadata | Disable submit; route to OTP; show validation/Auth/network errors         | Connected; Worker onboarding continues      | Contract + public E2E          |
| Verify                 | User, Worker | Verify/resend email OTP                            | Auth `verifyOtp` / `resend`                                                              | Email and six-digit code                                        | Active session and confirmed account                    | Challenge owner                                | Pending state; retry invalid/expired/provider failures                    | Connected                                   | Authenticated E2E pending      |
| Sign in/recovery       | User, Worker | Email/mobile login, Remember Me, reset             | Auth `signInWithPassword`; `auth-sign-in`; reset APIs                                    | Identifier/password or recovery email/code/password             | Rotating session and active fixed role                  | Account must be active                         | Persist or clear session as selected; reject wrong/inactive role          | Connected                                   | Storage tests only             |
| Admin login/MFA        | Admin        | Credentials and TOTP                               | Next Route Handlers → Supabase Auth/MFA                                                  | Email/password; factor ID/code                                  | SSR cookie session at required AAL                      | Active Admin; AAL2 where enabled               | Generic credential errors; never expose Auth internals                    | Connected                                   | Public route E2E; AAL2 pending |
